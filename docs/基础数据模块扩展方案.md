# 基础数据模块扩展方案

> 基于现有 BaseData 模块（工厂/车间/产线/工序）扩展物料与 BOM 管理
> 遵循现有代码模式和规范，不做架构变更

---

## 一、现有规范速查

| 方面 | 规范 |
|------|------|
| Entity 基类 | `BaseFullEntity`（Id + CreationTime + 软删） |
| 租户 | `[Tenant("AgentForge")]` |
| 表名 | `mes_` 前缀，`[SugarTable("mes_xxx")]` |
| 列名 | `[SugarColumn(ColumnName = "xxx")]`，全小写 |
| Repository 接口 | `ISqlSugarRepository<T> + ITransientDependency` |
| Repository 实现 | `SqlSugarRepository<T>`，双构造器 |
| DTO / Command / Query | `record` 类型 |
| 分页查询基类 | `PagedRequest`（Lzq.Core） |
| Service 接口 | `ITransientDependency` |
| Service 实现 | `ServiceBase` + `[OpenApiTag/Operation]` + `[RoutePattern]` |
| 映射 | `.Map<T>()`（Mapster） |
| 异常 | `UserFriendlyException` |
| 返回值 | `ApiResult<T>` |
| API 路由 | `/api/v1/mes/{resource}` |
| 模块注册 | `BaseModule` + `[DependsOn]` |

## 二、第一阶段：物料与 BOM 管理

### 2.1 新增实体（5 个）

| 实体 | 表名 | 说明 | 关键字段 |
|------|------|------|---------|
| `UnitOfMeasureEntity` | `mes_unit_of_measure` | 计量单位 | Code, Name, Category（数量/重量/体积/其他） |
| `MaterialTypeEntity` | `mes_material_type` | 物料类型 | Code, Name, ParentId（树形）, Level |
| `MaterialEntity` | `mes_material` | 物料主数据 | Code, Name, Spec, MaterialTypeId, UnitId, Weight, Volume, Status |
| `BomEntity` | `mes_bom` | BOM 头 | Code, Name, ProductId, Version, Status, EffDate, ExpDate |
| `BomItemEntity` | `mes_bom_item` | BOM 明细 | BomId, ItemId, Qty, ScrapRate, Sort, SubstituteIds |

### 2.2 新增枚举

| 枚举 | 值 | 用途 |
|------|----|------|
| `UnitCategoryEnum` | Count/Weight/Volume/Length/Other | 计量单位分类 |
| `MaterialStatusEnum` | Enabled/Disabled/Obsolete | 物料状态 |
| `BomStatusEnum` | Draft/Released/Obsolete | BOM 状态 |

### 2.3 数据结构关系

```
MaterialType (树形)
    └─ 1:N ── Material ── N:1 ── UnitOfMeasure
                   │
                   │ 1:N
                   ▼
               Bom ── 1:N ── BomItem ── 1:1 ── Material (ItemId)
```

### 2.5 Service 关键方法

| Service | 路由 | 方法 |
|---------|------|------|
| `UnitOfMeasureService` | `/api/v1/mes/unit-of-measure` | Page, Create, Update, Delete, BatchDelete, SelectList（下拉用） |
| `MaterialTypeService` | `/api/v1/mes/material-type` | Page, Tree, Create, Update, Delete, BatchDelete |
| `MaterialService` | `/api/v1/mes/material` | Page, Get, Create, Update, Delete, BatchDelete, SelectList |
| `BomService` | `/api/v1/mes/bom` | Page, Get, Create, Update, Delete, Copy（BOM 复制）, Release（发布） |

### 2.6 特殊逻辑

| 场景 | 处理方式 |
|------|---------|
| 物料编码唯一性 | Create/Update 时检查 `Code` 是否重复 |
| MaterialType 树形 | 含 `ParentId` 自引用，支持无限层级，删除检查是否有子节点 |
| BOM 版本管理 | 同一产品可多个 BOM 版本，同一时间只有一个 `Released` 状态的生效 |
| BOM 删除检查 | 被引用的 BOM 不允许物理删除，改为 Obsolete 状态 |
| BOM 复制 | 基于现有 BOM 复制一份新的 Draft 版本，版本号+1 |
| 物料删除检查 | 被 BOM 引用的物料不允许删除 |

## 四、技术备注

- DB 迁移：由 `SqlSugarProviderExtension.UseCodeFirst()` 自动建表，无手工迁移脚本
- 缓存策略：基础数据（物料/类型/单位）建议加 Redis 缓存，BOM 按需缓存
- 种子数据：基础计量单位（个/kg/m/L）、基础物料类型（原材料/半成品/成品）等通过 `BaseSeedData` 预置
