# 离散型 MES 功能大纲

> 目标：构建一个简单、功能齐全的离散制造执行系统（MES）
> 当前版本：v1 — 功能规划阶段

---

## 一、模块总览

```
┌─────────────────────────────────────────────────────────┐
│                     Lzq.MES                              │
├─────────────┬───────────┬──────────┬─────────┬─────────┤
│  基础数据   │  设备管理 │ 质量管理 │ 工单管理 │ 仓库管理 │
│  (BaseData) │ (Equipment)│   (QA)  │(WorkOrder)│(Inventory)│
├─────────────┼───────────┼──────────┼─────────┼─────────┤
│  生产执行   │  采购管理 │ 销售管理 │ 看板报表 │  系统管理 │
│ (Production)│  (Purchase)│  (Sales) │(Dashboard)│  (Rbac) │
└─────────────┴───────────┴──────────┴─────────┴─────────┘
```

## 十一、分期实施路线

| 阶段 | 内容 | 预估工作量 |
|------|------|-----------|
| **P1 — 基础扩展** | 物料/物料类型/计量单位 + BOM + 产品 | BaseData 扩展 |
| **P2 — 仓库管理** | 仓库/库位 + 出入库 + 库存台账 + 盘点 | 新建 Inventory 模块 |
| **P3 — 伙伴管理** | 客户 + 供应商 + 员工（基础数据层） | BaseData 扩展 |
| **P4 — 生产执行** | 生产计划 + 排程 + 领料 + 完工入库 | 新建 Production 模块 |
| **P5 — 采购与销售** | 采购订单 + 收货 + 销售订单 + 发货 | 新建 Purchase + Sales 模块 |
| **P6 — 质量增强** | 检验标准 + 8D + 追溯 | QA 扩展 |
| **P7 — 设备增强** | 备件 + OEE + 分类 | Equipment 扩展 |
| **P8 — 看板增强** | 新增看板 + 驾驶舱 | Dashboard 扩展 |

## 十二、技术约束

| 方面 | 规范 |
|------|------|
| 数据库表前缀 | 基础数据：`mes_`，按模块：`mes_${module}_` |
| 实体基类 | `BaseFullEntity`（Id + 审计 + 软删） |
| 租户标注 | `[Tenant("AgentForge")]` |
| 模块依赖 | 新建模块依赖 `Lzq.Extensions.SqlSugar` + `Lzq.Core` |
| 跨模块引用 | 统一走 `Application.Contracts` 接口层，不跨 Domain |
| API 路由 | `/api/v1/mes/{resource}` |
