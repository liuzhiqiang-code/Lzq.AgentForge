# 工单管理操作手册

## 功能概述

工单管理模块用于管理和跟踪生产工单的全生命周期，包括工单创建、查询、状态跟踪、报工、完工等操作。生产计划员、车间主任和操作工日常工作中需频繁使用此模块。

## 操作入口

- **菜单路径**：生产管理 → 工单管理
- **权限要求**：计划员（全部权限）、车间主任（查询+状态变更）、操作工（查询+报工）
- **快捷入口**：工作台 → 我的工单

## 操作步骤

### 场景1：查询工单信息

**适用角色**：计划员、车间主任、操作工

**操作目的**：查看工单详情、进度、状态等信息

**操作步骤**：

1. 进入「生产管理 → 工单管理」页面
2. 在查询条件区域输入查询条件：
   - **工单号**：支持模糊查询，如输入 "WO-202605" 可查询所有 5 月工单
   - **状态**：选择 pending（待处理）/ processing（进行中）/ completed（已完成）
   - **操作员**：输入操作员姓名快速筛选
   - **日期范围**：选择创建时间范围
3. 点击「查询」按钮
4. 在结果列表中点击工单号，查看详细信息：
   - 工单基本信息（产品、数量、计划时间）
   - 当前状态和生产进度
   - 操作记录
   - 关联设备和人员

**注意事项**：
- ⚠️ 工单号格式为：`WO-YYYYMMDD-序号`，如 `WO-20260510-001`
- ⚠️ 查询时尽量输入精确条件，避免返回过多数据影响系统性能
- 💡 可点击列表表头进行排序（按创建时间、状态等）

---

### 场景2：生产报工（完成数量录入）

**适用角色**：操作工、班组长

**操作目的**：录入实际完成的产品数量，更新工单进度

**操作步骤**：

1. 在工单列表找到当前正在处理的工单，点击「报工」按钮
2. 在报工页面填写：
   - **完成数量**：本次实际完成的好品数量
   - **废品数量**：本次产生的废品数量（如有）
   - **工时**：本次操作花费的工时（小时）
   - **备注**：可填写异常说明（可选）
3. 确认信息无误后，点击「提交报工」
4. 系统自动计算并更新工单进度百分比

**业务规则**：
- 📌 完成数量 + 废品数量 ≤ 工单计划数量 - 已报工数量
- 📌 每次报工都会记录操作人和操作时间
- 📌 当累计完成数量达到计划数量时，工单状态自动变为"completed"

**注意事项**：
- ⚠️ 报工数据直接影响生产统计，请务必准确填写
- ⚠️ 发现报工错误需联系计划员进行修正，不可自行删除
- 💡 建议每班次结束前完成报工，避免数据积压

---

### 场景3：暂停/恢复工单

**适用角色**：车间主任、计划员

**操作目的**：因设备故障、物料短缺等原因暂停工单，问题解决后恢复

**暂停操作**：

1. 找到需要暂停的工单，点击「状态变更」
2. 选择「暂停」操作
3. 填写暂停原因（必填）：
   - 设备故障
   - 物料短缺
   - 质量问题
   - 其他（需详细说明）
4. 点击「确认暂停」

**恢复操作**：

1. 找到已暂停的工单，点击「状态变更」
2. 选择「恢复」操作
3. 填写恢复说明（可选）
4. 点击「确认恢复」

**业务规则**：
- 📌 只有"processing"（进行中）状态的工单可以暂停
- 📌 暂停的工单不会出现在操作工的"待处理工单"列表中
- 📌 暂停期间不计入计划工时

---

### 场景4：关闭工单

**适用角色**：计划员

**操作目的**：工单全部完成或取消后，正式关闭工单

**操作步骤**：

1. 确认工单满足以下条件：
   - ✅ 累计完成数量 ≥ 计划数量，或工单已取消
   - ✅ 所有报工记录已审核
   - ✅ 无未处理的异常记录
2. 找到目标工单，点击「关闭工单」
3. 确认关闭原因：
   - 正常完成
   - 订单取消
   - 其他（需说明）
4. 点击「确认关闭」

**注意事项**：
- ⚠️ 工单关闭后不可再报工或修改
- ⚠️ 关闭前请确认所有数据已准确录入
- ⚠️ 关闭操作需要计划员权限

## 常见问题 (FAQ)

**Q: 我找不到我的工单，怎么办？**
A: 请检查：
1. 查询条件是否设置正确（日期范围、状态等）
2. 是否有权限查看该工单（联系计划员确认）
3. 工单号是否输入正确（格式：WO-YYYYMMDD-序号）

**Q: 报工时提示"数量超出"，如何解决？**
A: 系统不允许报工数量超过剩余数量。请：
1. 检查已报工数量：工单详情 → 报工记录
2. 联系计划员确认工单计划数量是否正确
3. 如确实需超报，需计划员调整工单计划数量

**Q: 工单状态显示"pending"，为什么不能开始生产？**
A: "pending"表示工单已创建但尚未开始。需要：
1. 计划员将工单状态改为"processing"
2. 确认物料、设备、人员已准备就绪
3. 确认工艺路线和BOM已审核

**Q: 报工后发现数量填错了，如何修改？**
A: 报工记录不可自行修改，需：
1. 联系计划员或系统管理员
2. 提供工单号和错误说明
3. 由计划员在后台进行修正

## 业务规则

### 工单状态流转

```
created（已创建） → pending（待处理） → processing（进行中） → completed（已完成） → closed（已关闭）
                    ↓
                 paused（已暂停） → processing（恢复后）
```

### 权限矩阵

| 操作 | 计划员 | 车间主任 | 操作工 |
|------|--------|---------|--------|
| 查询工单 | ✅ | ✅ | ✅（仅自己负责的） |
| 创建工单 | ✅ | ❌ | ❌ |
| 修改工单 | ✅ | ❌ | ❌ |
| 状态变更 | ✅ | ✅（暂停/恢复） | ❌ |
| 报工 | ✅ | ✅ | ✅ |
| 关闭工单 | ✅ | ❌ | ❌ |

### 报工规则

- 每次报工需填写完成数量、废品数量、工时
- 完成数量 + 废品数量 ≤ 剩余数量
- 同一工单每日最多报工 10 次（防止误操作）
- 报工后系统自动计算进度：进度% = 累计完成数量 / 计划数量 × 100%
